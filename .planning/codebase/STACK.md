# Technology Stack

**Analysis Date:** 2026-03-31

## Languages

**Primary:**
- Python 3.x - Desktop application with Tkinter GUI, file operations, and document template processing

## Runtime

**Environment:**
- Python 3.x (Windows-focused, uses win32 APIs)

**Package Manager:**
- pip
- Lockfile: Not detected (single `requirements.txt` file present)

## Frameworks

**GUI:**
- Tkinter (built-in Python library) - Desktop application UI with widgets, dialogs, and scrolled text areas

**Document Processing:**
- docxtpl - Word document template rendering and population

**Build/Packaging:**
- PyInstaller - Executable generation for Windows distribution (app.spec and ClickFolder.spec present)

## Key Dependencies

**Critical:**
- `pywin32` - Windows COM interoperability for shortcut creation via WScript.Shell
- `docxtpl` - Document template engine for generating Word documents (.docx files)
- `pyperclip` - Clipboard management (copy folder paths to clipboard)
- `pyinstaller` - Build tool for creating standalone Windows executables

**Standard Library (No external package):**
- `tkinter` - GUI framework (built-in with Python)
- `pathlib` - Path manipulation and validation
- `shutil` - File system operations (copy and delete)
- `os` and `stat` - OS-level utilities for file permissions
- `datetime` - Timestamp generation for logging

## Configuration

**Environment:**
- Configured via `config.py` with hardcoded defaults:
  - `DEFAULT_MARKETING_TEMPLATE` - Source template for marketing folder (M: drive path)
  - `DEFAULT_WORK_TEMPLATE` - Source template for work folder (M: drive path)
  - `DEFAULT_BD_TARGET` - Destination for BD/marketing copies (V: drive)
  - `DEFAULT_WORK_TARGET` - Destination for work folder copies (W: drive)
  - `FOLDER_TO_DELETE` - Folder name to remove post-copy ("1 Marketing")

**Build:**
- `app.spec` - PyInstaller configuration for main application executable
- `ClickFolder.spec` - PyInstaller configuration for alternate executable

## Platform Requirements

**Development:**
- Python 3.x
- Windows OS (dependency on win32com.client for shortcut creation)
- Network access to M:\, V:\, and W:\ network drives

**Production:**
- Windows OS (standalone executable generated by PyInstaller)
- Network access to M:\, V:\, and W:\ drives for templates and target locations
- User must have appropriate file permissions on shared drives

## Deployment Model

**Packaging:**
- Standalone Windows executable (console=False, windowed mode)
- Generated via PyInstaller with UPX compression enabled
- No external runtime required for end users

**Distribution:**
- Single executable file (`app.exe` from app.spec)
- Alternative packaged variant (`ClickFolder.exe` from ClickFolder.spec) exists
