## Mocking Patterns (Recommended Approach)

**Pattern:** Callback function pattern used throughout

The codebase passes a `log_func` callback to all utility functions for logging:

```python
def copy_folder(src, dst, log_func):
    # ...
    log_func(f"Copied from {src} to {dst}", "success")

def delete_folder(folder_path, log_func):
    # ...
    log_func(f"Deleted folder: {folder_path}", "success")
```

This pattern supports testing by allowing mock logging functions to be passed:

```python
# Example test pattern (if tests were implemented):
mock_log_calls = []

def mock_log(message, level="info"):
    mock_log_calls.append((message, level))

copy_folder(source_path, dest_path, mock_log)
assert len(mock_log_calls) > 0
assert any("Copied from" in call[0] for call in mock_log_calls)
```

**What Would Need Mocking:**
- File system operations (shutil, os)
- Path existence checks (pathlib.Path.exists())
- Windows COM objects (win32com.client)
- Tkinter GUI widgets for integration testing

**What Shouldn't Be Mocked:**
- Path operations (test with real temporary directories)
- Basic validation logic (can test with real conditions)

## Logging for Testing

**Current Logging Pattern:**

The codebase uses a callback-based logging pattern that facilitates testing:

```python
def write_log(self, message, level="info"):
    symbol = {"info": "🛠", "success": "✅", "error": "❌", "warn": "⚠️"}.get(level, "🛠")
    line = f"{symbol} {message}\n"
    self.log_text.config(state='normal')
    self.log_text.insert(tk.END, line)
    self.log_text.see(tk.END)
    self.log_text.config(state='disabled')
    self.root.update()
```

**Testing-Friendly Design:**
- All operations accept `log_func` parameter
- Allows test code to provide mock logging that records calls
- No global logger state
- Makes testing of error conditions possible (capture error logs)
