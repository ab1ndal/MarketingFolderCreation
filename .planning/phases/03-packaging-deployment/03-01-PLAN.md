<!-- Key facts for the executor. No codebase exploration needed. -->

From app.py _generate_a250 (line 276):
```python
template_path = Path("templates/A250.docx")
```
This is a bare relative path — it resolves correctly in dev (cwd = project root) but breaks in a PyInstaller bundle where the exe runs from a temp-extracted directory or the bundle folder, not the source tree. The fix is to resolve via sys._MEIPASS when frozen.

PyInstaller --onedir produces a COLLECT block (not an EXE-only block).
The existing app.spec / ClickFolder.spec use EXE with a.scripts + a.binaries + a.datas embedded — that is the --onefile pattern. --onedir requires a separate COLLECT() call.

PyInstaller datas format: list of tuples (source_glob, dest_folder_in_bundle)
  ('templates/A250.docx', 'templates')   <- places A250.docx in bundle/templates/

sys._MEIPASS is set by PyInstaller at runtime to the bundle's extraction/root directory.
Standard pattern:
```python
import sys
from pathlib import Path

def _resource_path(relative: str) -> Path:
    base = Path(getattr(sys, '_MEIPASS', Path(__file__).parent))
    return base / relative
```
Then: template_path = _resource_path("templates/A250.docx")

win32com hidden imports required for pywin32 to work in a bundle:
  - win32com.shell
  - win32com.shell.shell
  - win32timezone
  (win32com.client is imported directly; the shell submodule is loaded via COM dispatch at runtime and not caught by static analysis)

PyQt6 platform plugin: PyInstaller's PyQt6 hook handles qwindows.dll automatically via
hook-PyQt6.Qt6.Widgets. No manual binaries entry needed for standard PyQt6 install.

docxtpl bundles cleanly via PyInstaller's standard hooks (it uses lxml/jinja2 which are well-supported).

pyperclip: pure Python, no native extension — bundles without special treatment.

robocopy: Windows system binary at C:\Windows\System32\robocopy.exe — already on PATH on all Windows machines. DO NOT bundle it.

def _resource_path(relative: str) -> Path:
    """Resolve a resource path that works both in dev and in a PyInstaller bundle.

    In a bundle, sys._MEIPASS is set to the extraction root.
    In dev, falls back to the directory containing app.py.
    """
    base = Path(getattr(sys, '_MEIPASS', Path(__file__).parent))
    return base / relative
