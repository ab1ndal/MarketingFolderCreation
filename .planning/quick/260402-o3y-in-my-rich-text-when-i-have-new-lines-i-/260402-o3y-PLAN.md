<objective>
Fix inconsistent font in the Quill rich text editor: new lines switch from
Calibri Light 10pt to Calibri 11pt.

Purpose: When the user presses Enter in the Quill editor, Quill creates a new
`<p>` element. If Quill's internal default format does not match the CSS
override, Quill may inject a differing inline style (or lack of style) on
new paragraphs, causing the browser to fall back to its default font. Setting
the Quill editor's programmatic default format to match the CSS ensures every
paragraph — new or existing — renders in Calibri Light 10pt.

Output: assets/editor.html updated so the Quill instance is initialized with
Calibri Light / 10pt as its active default format for all new content.
</objective>

  <action>
    In assets/editor.html, update the `<script>` block that initialises Quill.

    The Quill snow theme does not know about 'Calibri Light' or '10pt' as named
    formats by default, so the approach is:

    1. **Register a custom whitelist for font and size** so Quill treats
       'calibrilight' as a valid font format value and '10pt' as a valid size.

    2. **After creating the Quill instance**, call `quill.format()` to set the
       active selection format to the desired defaults. This ensures the cursor
       (and any new paragraph created by Enter) starts with Calibri Light 10pt.

    Replace the existing `<script>` block with:

    ```html
    <script src="quill.js"></script>
    <script>
      // Register 'calibrilight' as a whitelisted font value
      var Font = Quill.import('formats/font');
      Font.whitelist = ['calibrilight'];
      Quill.register(Font, true);

      // Register '10pt' as a whitelisted size value
      var Size = Quill.import('attributors/style/size');
      Size.whitelist = ['10pt'];
      Quill.register(Size, true);

      var quill = new Quill('#editor', {
        theme: 'snow',
        modules: { toolbar: '#toolbar' }
      });

      // Apply default format so new paragraphs inherit Calibri Light 10pt.
      // quill.format() sets the format at the current selection/cursor
      // which Quill uses as the default for new content.
      quill.format('font', 'calibrilight');
      quill.format('size', '10pt');

      function getContent() { return quill.root.innerHTML; }
      function setContent(html) { quill.root.innerHTML = html; quill.update(); }
    </script>
    ```

    Also update the `.ql-editor` CSS selector in the `<style>` block to use
    the Quill font class approach for the container fallback. Add this rule
    inside the `<style>` block so the registered font class maps to the
    font-family:

    ```css
    /* Map registered Quill font value to actual font-family */
    .ql-font-calibrilight {
      font-family: 'Calibri Light', Calibri, sans-serif !important;
    }
    /* Ensure base editor and all children always use Calibri Light 10pt */
    .ql-editor,
    .ql-editor * {
      font-family: 'Calibri Light', Calibri, sans-serif;
      font-size: 10pt;
    }
    ```

    Keep the rest of the file (toolbar HTML, dark theme CSS) unchanged.

    Note: The quill.snow.css already has `.ql-container, .ql-editor, .ql-editor *`
    with `font-family: 'Calibri Light' !important; font-size: 10pt !important` from
    a prior change. The inline CSS in editor.html adds the `.ql-font-calibrilight`
    class mapping that Quill adds to `<span>` elements when the font format is set.
    This means both the CSS fallback AND Quill's own inline class will resolve to
    Calibri Light 10pt, eliminating any mismatch on new lines.
  </action>
