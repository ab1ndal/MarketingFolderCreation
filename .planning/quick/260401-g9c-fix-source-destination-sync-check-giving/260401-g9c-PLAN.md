<objective>
Fix the false-positive "Source and destination are already in sync (no files copied)" message that appears when creating a new destination folder.

Purpose: Robocopy exit code 0 is ambiguous — it means "nothing was copied" whether that is because everything was already in sync OR because the source template was empty. The current code cannot tell these apart, producing a misleading log message that makes users think the copy was skipped. The fix tracks whether the destination existed before the robocopy call, allowing the message to be accurate in both cases.

Output: Updated `operations/copy_ops.py` with correct exit-code-0 handling, updated tests.
</objective>
